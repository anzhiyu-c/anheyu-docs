## 🚀 快速开始

`安和鱼应用系统`是一个基于 **[Go](https://go.dev/) + [Vue3](https://vuejs.org/)** 构建的现代化内容管理与分享平台。系统采用前后端分离架构，后端使用 Go 语言提供高性能的 API 服务。

<Callout type="info" title="系统要求">
  在开始之前，请确保您的系统满足以下要求： 内存大于等于512MB，硬盘大于等于1GB，CPU大于等于1核。
</Callout>

## 🏗️ 技术架构

<Accordions type="single">
  <Accordion title="为什么要使用 GO作为后端？而不是Java或者 Rust">
    - Go 语言的性能和效率非常出色，适合构建高性能的 API 服务 - 运行时仅需`10MB`左右，非常轻量级 -
    编译速度非常快，支持多平台编译，支持静态编译，开箱即用 - 支持多线程，支持并发，支持异步，支持协程 -
    Java内存占用过大，Go有许多好用的标准库，Rust相对较少，当我需要一个功能的时候，我可以使用Go快速实现，Go的性能表现已经足够。
  </Accordion>
  <Accordion title="为什么选择使用Vue3作为前端？而不是React/Nuxt.js/Next.js">
 - Vue3的性能和效率非常出色，Vue3的生态系统非常成熟，有很多优秀的库和工具可以使用。
 - 我需要写大量的动画，我不喜欢写内链样式，Vue3的 Composition API 非常适合我，但是React的CSS写的我很累，虽然同样支持CSS in JS，但是React的CSS in JS写的我很累。
 - Nuxt与Next等最大的优势是支持SSR，但是我的后端已经使用Go的静态嵌入实现了SSR，已经有了非常优秀的 SEO 能力。
 - 为什么官网却使用Next.js？因为官网需要非常优秀的 SEO 能力，并且系统不大，Next.js的 SEO 能力非常出色。
 - 我希望能有更多的人能够通过本项目学习到编程开发，Vue更易上手，这有利于后续集成主题开发者以及生态。
  </Accordion>

  <Accordion title="支持哪些数据库？">
 - **MySQL** - 关系型数据库
 - **PostgreSQL** - 开源对象关系数据库（推荐）
 - **SQLite** - 轻量级嵌入式数据库
  </Accordion>
</Accordions>

### 架构模式

- **前后端分离** - API 驱动的现代化架构
- **RESTful API** - 标准化的接口设计
- **容器化部署** - 支持 Docker 和 Docker Compose
